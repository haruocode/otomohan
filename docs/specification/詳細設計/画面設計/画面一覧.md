# 画面設計

## 前提

- Webアプリ（SPA）
- ユーザーとおともはん（話し相手）の2ロール
- 音声通話（1対1）
- ポイント制（1分45pt）
- 認証あり（ログイン必須）
- プロフィール閲覧、通話リクエスト、通話中画面、履歴表示など
- 運営管理画面（将来的に）も考慮

---

# 🔷 全画面一覧（完成版）

利用者（User）、おともはん（Otomo）、共通画面、運営画面に分けて整理します。

# 🚻 共通画面（User / Otomo 共通）

| 画面ID | 画面名                          | 説明                                   |
| ------ | ------------------------------- | -------------------------------------- |
| C-01   | ログイン画面                    | メール/パスワード、SNSログイン（任意） |
| C-02   | 会員登録画面                    | 名前・メール・パスワード・確認コード   |
| C-03   | プロフィール編集                | 名前、アイコン、自己紹介               |
| C-04   | 設定画面                        | 通知設定、音声テスト、ログアウト       |
| C-05   | ヘルプ / FAQ                    | 利用ガイド                             |
| C-06   | 利用規約 / プライバシーポリシー | 静的ページ                             |
| C-07   | ブロックリスト管理              | 特定のおともはん/ユーザーをブロック    |

# 👤 利用者（User）向け画面

| 画面ID | 画面名                        | 説明                                                 |
| ------ | ----------------------------- | ---------------------------------------------------- |

| U-06   | ウォレット（ポイント残高）    | 保有ポイント＋チャージボタン                         |
| U-07   | ポイントチャージ画面（将来）  | クレカ / PayPay / Stripe など                        |
| U-08   | 通話履歴一覧                  | 過去の通話を一覧表示                                 |
| U-09   | 通話履歴詳細                  | 履歴詳細（開始時間、終了理由、課金明細）             |
| U-10   | マイページ                    | プロフィール・通話履歴・ウォレットへの導線           |
