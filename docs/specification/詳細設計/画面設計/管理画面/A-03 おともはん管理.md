# A-03 おともはん管理

A-02（一般ユーザー管理）とは異なり、**「おともはん（声の提供者）を管理・審査・監視するための管理画面」** です。

サービス品質と安全性に直結するため、より詳細な情報管理が必要になります。

---

# 1. 画面概要

管理者が、おともはん（通話相手として登録している提供者）を管理するための画面。

目的：

- プロフィール・通話品質・実績の管理
- 審査／承認（A-04 と連携）
- 稼働状況の確認
- レビュー状況（評価）
- アカウント凍結 / 強制離脱
- 不適切行動 / 通報の対応

一般ユーザー以上に厳格な監査が必要。

## ③ おともはん詳細画面（モーダル or 別ページ）

### 1) 基本情報

- ID：`otomo_001`
- 氏名
- 表示名（ユーザーへ見せる名前）
- メールアドレス
- プロフィール文
- プロフィール画像
- 登録日
- 最終ログイン

---

### 2) アカウント状態

```
状態：承認済み（稼働可能）
[ 一時停止にする ]  [ 凍結する ]  [ 解除 ]
```

状態変更時は必ず理由入力モーダル：

```
変更理由：
[                               ]
[ 実行する ]
```

→ A-10 管理ログに記録される

---

### 3) 通話実績（重要）

- 通話数
- 通話合計時間
- 直近30日の通話グラフ
- 人気時間帯（ヒートマップ）

例：

```
通話数：350回
通話合計時間：890分
平均通話時間：8.4分
```

→ クリックで個別通話へ（A-06）

---

### 4) 収益状況（将来）

- 累計収益
- 月ごとの収益
- 直近の収益トレンド

※ A-05（ポイント管理）とは別の集計

# 5. セキュリティ

- おともはんは外部公開されるため、より厳重な監査が必要
- 管理画面の操作は A-10 で全てログに残す
- 一時停止や凍結には二段階確認（MFA推奨）
- プロフィールやレビュー改ざんは禁止（閲覧のみ）

---

# 6. デザイン仕様

- テーブル中心で情報量が多い
- 状態タグで視認性を高める
- 通話グラフや評価分布をカード表示
- ダーク/ライトどちらでも可
- 管理画面共通の UI ガイドラインに準拠
