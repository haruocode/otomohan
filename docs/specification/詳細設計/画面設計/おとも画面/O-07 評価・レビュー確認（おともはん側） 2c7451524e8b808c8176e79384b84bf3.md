# O-07 評価・レビュー確認（おともはん側）

ユーザーが通話後に投稿したレビュー（5 段階評価・コメント）をおともはんが確認するための画面です。運営側の品質管理にも関わるため、UI と情報構造は丁寧に整理しています。

---

# 1. 画面概要

おともはんが「ユーザーからの評価（星）とレビューコメント」を閲覧する画面。

目的：

- 自身のパフォーマンスの振り返り
- リピート率向上のための改善
- 運営側からの注意通知（任意）との連携

レビューは匿名化（ユーザー ID を直接見せない）を基本とする。

---

# 2. 画面構造（UI）

---

## **ヘッダー**

| 要素 | 内容     |
| ---- | -------- |
| 左   | ＜ 戻る  |
| 中央 | レビュー |
| 右   | なし     |

---

# 3. メインコンテンツ

---

## ① 総合評価（上部サマリー）

中央上に大きく：

```
★ 4.6 / 5.0
```

サブ情報：

- 評価件数： 128 件
- 通話総数： 350 回
- リピート率： 42%（任意表示）

棒グラフで「星 1〜5 の割合」を表示：

```
★★★★★  70%
★★★★☆  20%
★★★☆☆   7%
★★☆☆☆   2%
★☆☆☆☆   1%
```

---

## ② 直近レビュー（最新順）

レビューカードを縦並びで表示。

### レビューカード構成：

```
[星4]   2025/01/30
たろう さん（匿名加工 or 固定名）

コメント：
「丁寧に話してくれて嬉しかったです！」

通話時間：12分
```

- 星評価：★ アイコン（ピンク/ブルー）
- 日付（レビュー送信日時）
- ユーザー名は匿名化 _「ユーザー A」「ユーザー B」_ でも良い
  （プライバシー設計による）
- コメントは 3 行まで表示し、それ以上は「続きを読む」で展開

---

## ③ レビュー検索・フィルター（任意拡張）

画面上部、または右上にフィルターアイコンを配置し、以下が可能：

- 星数で絞り込み（★5 のみ、★1〜2 のみなど）
- 通話時間で並び替え（長い通話順）
- レビューあり／なし
- 指摘が多いレビュー（低評価）

例：

```
[フィルター] 星5 / 星4 / 星3以下 / コメントあり
```

---

## ④ 特別レビュー / 運営からの注意（任意）

低評価が続いた場合など、運営が通知を送る仕組みも拡張可能：

```
⚠ 運営からの注意
最近、低評価レビューが複数見られます。対応を改善しましょう。
```

※ この要素は将来導入のオプション。

---

## ⑤ レビュー詳細（タップ遷移）

レビュー 1 件をタップすると O-08（レビュー詳細）を表示できる構造にもできる。

---

# 4. 空状態（レビューがまだ無い場合）

```
まだレビューはありません
通話後、ユーザーからの評価がここに表示されます
```

＋シンプルなイラスト（任意）

---

# 5. API 連動

| 用途                     | API                        |
| ------------------------ | -------------------------- |
| 総合評価の取得           | GET /otomo/reviews/summary |
| レビュー一覧取得         | GET /otomo/reviews         |
| レビュー詳細取得（任意） | GET /otomo/reviews/{id}    |
| 運営からの注意（任意）   | GET /otomo/alerts          |

### レビューのデータ構造例

```json
{
  "rating": 4,
  "comment": "楽しく話せました！",
  "createdAt": "2025-01-30T12:00:00Z",
  "duration": 720,
  "user": {
    "maskedName": "ユーザーC"
  }
}
```

# 6. デザイン仕様（既存 O 系と統一）

- ダークモード
- 星はピンク/ブルーで発光エフェクト
- カードは角丸・黒背景
- 棒グラフもネオンブルーで統一
- コメントは可読性重視で白テキスト
