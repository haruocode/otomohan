# O-01 おともはんホーム（待機画面）

**“おともはんが待機し、自分のステータスを管理し、通話リクエストを受け取る”**

という目的に最適化した構造で設計しています。

---

# 1. 画面概要

おともはん（話し相手側）がログイン後に最初に到達する画面。

この画面では：

- ステータス（オンライン／離席中／通話中）切り替え
- 自分のプロフィール確認
- 待機中にユーザーからの通話リクエストを受信
- 報酬（ポイント）残高の確認
- 過去履歴や設定へ遷移

といった基本操作を行う。

**「通話が来るまでの待機画面」** が主目的。

---

# 2. 画面構造

## **ヘッダー**

| 要素 | 内容 |
| --- | --- |
| 左 | 設定アイコン（設定 C-04 へ） |
| 中央 | おともはんホーム |
| 右 | 通知ベル（未読件数バッジ任意） |

---

# **メインコンテンツ**

## ① プロフィールカード（上段）

- 丸い大きめのプロフィール画像（80〜100px）
- 名前（例：さくら）
- ステータス表示
    - ● 緑：待機中（受付中）
    - ● 黄：離席中
    - ● 赤：通話中（自動で切り替わる）

ステータスはドロップダウンまたはトグル式で変更：

```
[待機中 ▼]
 - 待機中（受付可能）
 - 離席中（受付停止）
```

---

## ② ステータス説明エリア（小テキスト）

例：

```
「待機中」の場合、ユーザーからの通話リクエストを受け取ります。
「離席中」にするとリクエストが届きません。
```

---

## ③ 報酬（ポイント）残高カード

- 現在ポイント
- 日付別集計（任意）

例：

```
本日の報酬：1250 pt
累計報酬：43200 pt
```

- 「報酬履歴を見る」ボタン

※ おともはん側のポイントはユーザー側と別管理。

---

## ④ 通話履歴（最新 1〜2 件）

例：

```
・たろう さん / 21:30〜21:37（7分）
・みゆき さん / 20:10〜20:30（20分）
[通話履歴を見る >]
```

---

## ⑤ 待機状態ビジュアル（中心部）

着信が来ていないときの「待機中」を示すアニメーション的エリア。

例：

- 波紋アニメ風
- 「待機中…」ラベル
- 明るいピンク/ブルーのネオンリング

アプリの世界観を最も象徴する部分になる。

---

## ⑥ 通話リクエストを受信した場合（O-02へ遷移）

O-02（着信画面）へ自動で切り替わる。

---

## ⑦ フッター（ナビゲーション）

- ホーム
- 通話履歴（おともはん視点の履歴）
- 報酬（ポイント）
- 設定

ユーザー側 U 系とほぼ同じ構造だが、

**「報酬」タブが追加される点が違い。**

---

# 3. ステータス仕様

| ステータス | 意味 | ユーザー側表示 | 通話リクエスト受付 |
| --- | --- | --- | --- |
| 待機中（オンライン） | 受付可能状態 | 「今すぐ話せます」 | 〇 |
| 離席中 | 休憩・作業中 | 「現在離席中です」 | ✕ |
| 通話中 | 通話中 | 「通話中」 | 自動拒否 |

API 連動：

`PUT /otomo/status`

```json
{
  "status": "online" | "away" | "busy"
}
```

---

# 4. 画面遷移

```
O-01 おともはんホーム（待機）
   ↓ （通話リクエスト受信）
O-02 着信画面
   ↓
U-04 / O-04 通話画面
   ↓
U-05 / O-05 通話終了サマリー
```

---

# 5. API 連携

| 機能 | API |
| --- | --- |
| ステータス取得 | GET /otomo/me |
| ステータス変更 | PUT /otomo/status |
| 履歴一覧 | GET /otomo/calls |
| 報酬表示（任意） | GET /otomo/rewards |
| 着信通知受信 | WS（call_request） |

---

# 6. デザイン（既存 UI と統一）

- ダークモード
- ピンク/ブルーのネオンアクセント
- ステータスは丸いカラーインジケータ
- ボタンは丸角で光沢のあるグラデーション
- 待機中アニメーション（リング）は U 系と差別化ポイント
