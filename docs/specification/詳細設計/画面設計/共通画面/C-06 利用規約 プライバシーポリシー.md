# C-06 利用規約 / プライバシーポリシー

# 1. 画面概要

利用規約・プライバシーポリシーの内容をユーザーが閲覧する画面。

C-04 設定画面から遷移し、スクロールして内容を確認できる。

- 特にアカウント削除や課金周りを扱うアプリなので、
  **説明の読みやすさ・信頼感** が重要。
- WebView で HTML を読み込む形式にし、
  スクロールしやすいレイアウトを採用。

---

# 2. 画面構造（UI）

## **ヘッダー**

| 要素 | 内容                                 |
| ---- | ------------------------------------ |
| 左上 | ＜ 戻る                              |
| 中央 | 利用規約 または プライバシーポリシー |
| 右上 | なし                                 |

---

## **メインコンテンツ**

### ① タイトル

- 「利用規約」または「プライバシーポリシー」
- H1 ラベルを太字で表示（WebView 上）

---

### ② 文面表示領域

- アプリ内 WebView または長文スクロールビュー
- 文字サイズ：14〜15px
- 行間：1.6
- 背景：ダークグレー（#111〜#161616）

---

### ③ 戻る動作

- 閉じると設定画面（C-04）へ戻る

---

# 3. 表示方式の選択肢（推奨含む）

## **方式 A：API で取得 → アプリ内 WebView で表示（推奨）**

- `GET /settings` にて以下を取得：

```json
{
  "terms_url": "https://example.com/terms",
  "privacy_url": "https://example.com/privacy"
}
```

- 外部 HTML を読み込んで表示
- 利点：運営側が文面を差し替えてもアプリ更新が不要

---

## **方式 B：文面をアプリ内に埋め込む**

- 変更時はアプリ更新が必要
- 安全だが柔軟性が低い

→ 現代のモバイルアプリでは **A 方式** が一般的。

---

# 4. デザインルール

- **ダークモード**
- 本文は白（#EEE）
- 見出しはピンク/ブルーのアクセント線
- スクロールバーは細く、背景になじませる
- 長文でも疲れない可読性優先 UI

# 6. API 連携

| 処理                | API                          |
| ------------------- | ---------------------------- |
| 利用規約URL取得     | GET /settings                |
| プライバシーURL取得 | GET /settings                |
| 閲覧ログ（任意）    | POST /analytics/viewed_terms |

※ アカウント作成時に「同意」チェックが必要なら

`accepted_terms_version` を user テーブルに保持。

# 8. 特記事項：通話アプリ特有の注意点

規約内に必ず追記すべき項目：

- 課金ポイントに関する説明
- 通話の監視・録音を行わない旨
- 不正行為（複数端末・ポイント不正）
- medasoup（SFU）利用に伴う接続情報の取り扱い
- 通信品質による切断について免責事項

必要であれば、運営視点での **規約本文のドラフト** も作成できます。
